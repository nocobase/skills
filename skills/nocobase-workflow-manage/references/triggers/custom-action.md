# 自定义操作事件

## 触发器类型

`custom-action`
请使用以上 `type` 值创建触发器，不要使用文档文件名作为 type。

## 适用场景
- 需要通过界面上的"触发工作流"按钮手动触发流程，而非由数据变更或绑定在内置操作按钮上触发。
- 内置的增删改操作无法满足业务需求，需要通过工作流定义一系列复杂操作逻辑。
- 需要在表单提交、记录行操作或全局按钮中绑定自定义工作流。

## 触发时机 / 事件
- 用户点击绑定了该工作流的"触发工作流"按钮时触发。
- 通过 API 调用 `workflows:trigger`（自定义上下文） / `<collection>:trigger`（绑定数据表上下文）操作并传入 `triggerWorkflows` 参数时触发。
- 支持同步和异步两种执行模式（由工作流的 `sync` 属性决定）。

## 配置项列表
| 字段 | 类型 | 默认值 | 必填 | 说明 |
| --- | --- | --- | --- | --- |
| type | number | 0 | 是 | 上下文类型：`0` 全局自定义数据（Global），`1` 单条记录（Single record），`2` 多条记录（Multiple records）。 |
| collection | string | - | 上下文类型为 1 或 2 时必填 | 触发数据所在数据表，格式为 `"<dataSource>:<collection>"`（如 `"posts"` / `"mysql:orders"` 等，数据源为主数据源时可省略 `dataSource`）。上下文类型为 `0`（全局）时可不填。 |
| appends | string[] | [] | 否 | 预加载关联字段路径（仅在绑定了数据表时生效）。 |

## 上下文类型说明

### 全局自定义数据（type = 0）
- 不依赖特定数据表，可在任意位置的按钮上使用。
- 触发时传入自定义 JSON 数据作为触发数据。
- 适用于不与特定记录关联的操作场景。

### 单条记录（type = 1）
- 必须绑定数据表，在表单、详情区块或表格行操作按钮中使用。
- 触发时会加载指定记录数据，并与表单提交数据合并。
- 支持通过 `appends` 预加载关联数据。

### 多条记录（type = 2）
- 必须绑定数据表，在表格区块的批量操作按钮中使用。
- 触发时通过 `filterByTk` 批量加载多条记录数据，触发数据为数组。

## 触发器变量
- `$context.data`：触发的数据。
  - 全局模式：用户提交的自定义 JSON 数据。
  - 单条记录模式：记录数据（含 `appends` 预加载的关联数据）。
  - 多条记录模式：记录数据数组。
- `$context.user`：触发操作的用户（脱敏后的用户信息）。
- `$context.roleName`：触发操作的用户角色名称。

## 同步执行与拦截
- 当工作流设置为同步执行时，流程会在请求中同步完成。
- 同步流程如果执行失败或被"结束流程"节点终止（状态为失败），将拦截请求并返回错误信息。
- 异步流程在请求完成后在后台执行，不会拦截请求。

## 界面操作配置

该事件需要在可以添加“触发工作流”的区块中添加按钮，并绑定工作流后，用户才能正常使用。API 调用则无需配置）。

## 示例配置

### 全局自定义数据
```json
{
  "type": 0
}
```

### 单条记录
```json
{
  "type": 1,
  "collection": "posts",
  "appends": ["category", "author"]
}
```

### 多条记录
```json
{
  "type": 2,
  "collection": "mysql:orders"
}
```

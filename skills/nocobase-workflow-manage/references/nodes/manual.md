# Manual Process

## Node Type

`manual`

## Node Description
Generates a to-do task, waits for designated personnel to process and submit the form before continuing the process (only available for asynchronous workflows).

## Business Scenario Example
Manually checking information before continuing the process.

## Configuration List
| Field | Type | Default | Required | Description |
| --- | --- | --- | --- | --- |
| assignees | array | [] | Yes | List of assignees. Elements can be user ID variable of user ID. Support only one user for now. |
| title | string | Node Title | No | Task title, supports variable templates. |
| schema | object | None | No | Task interface UI Schema (generated by the configuration interface). |
| forms | object | {} | No | Form configuration (generated by the configuration interface), including definitions for custom/add/update forms, etc. |

## Branch Description
Does not support branches.

## Example Configuration
```json
{
  "assignees": ["{{ $context.data.ownerId }}"],
  "mode": 0,
  "title": "{{ $context.data.title }} - Pending"
}
```

## Output Variables
The variable selector for this node is a tree array of `{ label, value, children? }`. At runtime, join the `value` segments with `.` and prepend `$jobsMapByNodeKey.<nodeKey>`.

- This node exposes variables only when `config.forms` contains one or more configured forms.
- Each form key becomes a child root under the node, and that form root expands to the form collection's fields.
- Example reference pattern: `{{$jobsMapByNodeKey.manual_review.<formKey>.<fieldName>}}`.
- If `forms` is empty, this node provides no output variable tree.

# CC

## Node Type

`cc`
Use the above `type` value when creating the node; do not use the documentation filename as the type.

## Node Description
Sends a CC task/notification to specified users, used for viewing or informing only.

## Business Scenario Examples
Sending a CC to relevant stakeholders for information after an approval is completed.

## Configuration Items
| Field | Type | Default | Required | Description |
| --- | --- | --- | --- | --- |
| users | array | [] | Yes | List of recipients. Elements can be user IDs or user filter objects (filter structure). |
| ccDetail | string | None | Yes | UID for the CC detail interface configuration (generated by UI). |
| title | string | Node Title | No | Task title, supports variable templates. |
| notifications | array | [] | No | Notification configurations (generated by UI). |

## Branch Description
Does not support branching.

## Example Configuration
```json
{
  "users": ["{{ $context.data.ownerId }}"],
  "ccDetail": "cc-ui-uid",
  "title": "{{ $context.data.title }} - CC"
}
```
